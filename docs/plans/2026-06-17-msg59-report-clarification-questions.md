## A. 报表内容规范（对应 Plan B-2，**最高优先级 —— 不答复则后端引擎无法启动**）

| # | 问题 | 为什么需要 |
|---|---|---|
| A-1 | **报表清单**：§5.9 需生成哪些监管报表？请逐张列出（报表名称 / 编号 / 用途）。 | 决定领域模型与生成器数量；当前两份文档 0 张报表清单。 |
| A-2 | **各报表字段与口径**：每张报表的列/字段、计算口径（如「成功率」分母含哪些状态）、汇总维度（按日/机构/报文类型？）。 | 报表生成需精确字段映射；口径错=报表错。 |
| A-3 | **报送频率与触发**：各报表是日报/周报/月报/实时？定时自动生成还是人工触发？数据截止时点（T / T+1）？ | 决定调度与数据快照边界。 |
| A-4 | **数据范围**：报表数据取自哪些业务（供应链融资 / 企业查询 / 通用）？是否限本机构、是否含跨机构汇总？ | FEP 现有数据源 = `integration/tracking/*` 业务专表，须确认覆盖面是否足够。 |
| A-5 | **输出格式适用**：Excel / PDF / XML 三种格式各用于哪类报表？是否每张报表都要三格式，还是按报表区分？ | 决定 writer 工作量；避免全量三格式的过度实现（YAGNI）。 |
| A-6 | **报表模板「自定义」粒度**：PRD「支持自定义报表格式」指——(a) 固定模板集（机构从预置模板选），还是 (b) 用户可视化自定义模板（拖拽/DSL）？ | (a) 与 (b) 工作量差一个数量级，须先定。 |

## B. 报送通道（对应 Plan B-3）

| # | 问题 | 为什么需要 |
|---|---|---|
| B-1 | **报送通道**：生成的报表经 TLQ 报送至 HNDEMP 时，是**复用既有报文报送链**（若是，对应哪个报文号？如 1101 外联机构数据报送报文），还是**新建独立文件报送通道**？ | 决定是否复用既有 outbound 流水线还是新建。 |
| B-2 | **报表签名要求**：报送的报表是否需国密 SM2 签名 / SM4 加密？（若需→FEP 侧走 ⛔ 安全评审网，工期与评审流程不同。） | 决定是否触发安全禁入区评审 + 工期。 |
| B-3 | **在线/离线报送**：设计文档 §4.3 提「支持在线和离线报送方式」——离线报送的具体形态（落盘 NAS/SFTP？人工导出？）与触发条件？ | 离线路径可能涉及额外文件交换基础设施。 |
| B-4 | **报送日志要求**：报送日志需记录哪些字段、保留多久、是否需对账/重发能力？ | 决定 `report_generation_log` / 报送日志表 schema。 |

## C. 原型 / UI（对应 Plan B-1）

| # | 问题 | 为什么需要 |
|---|---|---|
| C-1 | **原型补充时间表**：PRD 明示「§5.9 原型中无专门页面，需补充原型设计」——原型预计何时提供？ | 管理 Web（Phase C）在原型出来前不启动（无原型的 UI 必返工）。 |
| C-2 | **角色与操作**：报表功能面向哪些角色（系统管理员 / 业务人员 / 审计人员）？各角色可做哪些操作（生成 / 下载 / 报送 / 配置模板）？ | 决定权限矩阵（参照 §5.9.1 运维监控权限矩阵风格）。 |

## D. 验收口径

| # | 问题 | 为什么需要 |
|---|---|---|
| D-1 | **报表正确性验收**：每张报表如何验收正确（是否有标准样张 / 对照数据 / 监管校验工具）？ | FEP 侧 TDD 须用「真实样张」断言生成产物（镜像红线 `xsd_compliance_fix_real_validator_on_sut`），无样张无法写有效测试。 |
